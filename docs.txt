## 🕹️ The Refined 3-Step Flow

### **1. Initialize the Game (The Hardware Handshake)**

* **Memory Pool Setup:** Allocate **192MiB** via `VirtualAlloc`.
* **Partitioning:** * `000-064MB`: **Static Assets** (Textures/Sounds).
* `064-128MB`: **The Backbuffer** (ARGB pixel array for the 165Hz display).
* `128-144MB`: **The Z-Buffer** (Stores depth for every pixel to handle sprite clipping).
* `144-192MB`: **The World State** (Map grid, entity positions, pathfinding nodes).




* **High-Frequency Clock:** Call `QueryPerformanceFrequency`.
* Calculate `Target_Ticks = Frequency / 165`.


* **Window Creation:** Open a window using `StretchDIBits`. This allows the CPU to "blast" your 128MB backbuffer directly to the GPU's front buffer.

### **2. Run the Game (The 165Hz "Heartbeat")**

This loop must be razor-sharp. If a frame takes longer than 6.06ms, the game feels "heavy."

* **Logic & Physics:**
* **Entity Pulse:** Iterate through the entity list. Check `Dist(Player, Entity)`. If < 0.2, trigger collision.
* **Update Map/Player:** Apply $Velocity \times DeltaTime$. Check the Map Grid. If the target cell is `> 0` (a wall), stop movement on that axis.


* **The "Add-On" Rendering (The Doom Effect):**
* **DDA Raycasting:** Cast a ray for every horizontal pixel.
* **Z-Buffer Shading:** For every pixel rendered, write the distance to the Z-Buffer.
* **Floor/Ceiling Texturing:** Once a wall is hit, use the remaining vertical screen space to calculate "Floor Projections." Map a texture to the ground based on the player's height and tilt.
* **Sector Lighting:** Every map cell has a `Light_Level (0.0 to 1.0)`. Multiply the final pixel color by `(Cell_Light / Distance)`. This allows for flickering lights and dark corridors.


* **HUD Overlay:** Blit the `ui_hud_base.bmp` and the `spr_gun` over the 3D view.
* **The Wait:** Use a `while` loop with an `_mm_pause()` (x86 assembly) until the `Target_Ticks` are met. This keeps the 165Hz perfectly stable.

### **3. End the Game (Termination)**

* **Break Loop:** On "Exit" click, stop the music thread.
* **Clean Release:** `VirtualFree` the 192MiB pool.
* **Return 0:** Signal Windows that the process ended successfully.

## 🖥️ CLI_ALT (The Terminal Ghost)

* **Mode:** `STD_OUTPUT_HANDLE` only. No Windows GUI.
* **Clock:** `Sleep(100)` (10Hz refresh).
* **Visuals:** Uses a character ramp for depth: `@#W#*+=-:. `
* **Logic:** A simplified version of the raycaster that outputs a single `char` per screen column instead of a pixel.

## 🛠️ Compilation BAT File (`build.bat`)

This script compiles both the high-end game and the CLI version using the MSVC compiler.

```batch
@echo off
if not exist "bin" mkdir bin

:: --- Compile Main Doom-Like ---
:: /O2: High optimization /Oi: Enable intrinsics /link: Windows Libs
cl.exe /nologo /Zi /O2 src/main.c /Febin/game_165hz.exe /link user32.lib gdi32.lib winmm.lib

:: --- Compile CLI Version ---
cl.exe /nologo /O2 src/cli_alt.c /Febin/game_cli.exe

echo Build Complete. Running Main Game...
pause

```

## 🧠 Technical "Pro-Tips" for Copilot/You

* **Assembly Tip:** If the compiler tries to "unroll" your raycasting loop and ruins the cache performance, use `#pragma optimize("", off)` or wrap the inner loop in a naked `__asm` block for the DDA steps.
* **Fixed Point Math:** Since you're on a single core, consider using **Fixed Point Arithmetic** (shifting integers) instead of `float` to keep the CPU's FPU (Floating Point Unit) from bottlenecking.
