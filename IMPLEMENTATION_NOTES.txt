// ===== HOW TO USE THE GRAPHICS =====

// The game looks for graphics in: src/GRAPHICS_SOUNDS/GRAPHICS/
// Three entity texture files can be loaded:
//
// 1. BOSS_ENTITY.png/bmp         - Big scary boss textures
// 2. NORMAL_ENTITIY.png/bmp      - Normal enemy textures
// 3. RED_AGRESSIVE_ENTITIY.png/bmp - Red aggressive enemy textures

// IMPORTANT: Convert PNG to BMP!
// ================================
// Run this to convert all PNG files to BMP format:
//   powershell -ExecutionPolicy Bypass -File convert_images_to_bmp.ps1
//
// Or manually convert using any image editor (Photoshop, GIMP, Paint, etc):
//   Right-click image → Save As → Format: BMP (24-bit or 32-bit)

// Why BMP instead of PNG?
// - BMP: Simple format, easy to parse in C, no external libraries needed
// - PNG: Complex compression, requires zlib+libpng or external decoder
// - Conversion is lossless, so image quality is preserved

// ===== TEXTURE LOADING FALLBACK CHAIN =====

// If graphics files are missing, the game won't crash!
// Instead, it falls back to procedurally generated textures:
//
// Priority Order:
// 1. Try PNG file  (won't load without PNG library)
// 2. Try BMP file  (NOW WORKS with converted images)
// 3. Generate procedural pattern (ALWAYS works as backup)
//
// Procedural patterns are:
// - 256x256 pixels (large enough to see details)
// - Distinctive for each entity type (can tell them apart)
// - Generated algorithmically at startup (fast!)
